# Development and GitHub Activity Standard

Development may be prepared locally in advance, but every committed milestone must include genuine work performed on its actual commit date.

Before committing a milestone:

1. Review its implementation.
2. Make a meaningful improvement.
3. Run the relevant automated tests.
4. Inspect the generated outputs or reconciliation results.
5. Update documentation only for completed capabilities.
6. Commit using the current real date.
7. Push the milestone branch and merge a reviewed pull request.

The project does not:

- Alter author or committer dates
- Change the computer’s system date
- Create empty or meaningless commits
- Rebase solely to change contribution dates
- Claim unfinished functionality
- Delay previously created commits merely to simulate daily activity

Issues and pull requests are useful development evidence only when they correspond to real work.
